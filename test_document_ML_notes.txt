CHAPTER 1: INTRODUCTION TO MACHINE LEARNING
==========================================

Machine Learning (ML) is a branch of Artificial Intelligence that enables computers to learn from data and improve their performance on tasks without being explicitly programmed for each task. The term was coined by Arthur Samuel in 1959.

The core idea behind machine learning is: Instead of writing rules manually, we feed data to an algorithm and let it figure out the rules on its own.

Traditional Programming:
  Input + Rules → Output

Machine Learning:
  Input + Output → Rules (learned automatically)

WHY IS MACHINE LEARNING IMPORTANT?
- Handles complex problems too difficult to code manually (e.g., face recognition)
- Adapts and improves with more data over time
- Discovers hidden patterns in large datasets
- Powers modern AI applications: search engines, recommendation systems, virtual assistants

CHAPTER 2: TYPES OF MACHINE LEARNING
==========================================

2.1 SUPERVISED LEARNING
-----------------------
In supervised learning, the model is trained on labeled data. Each training example has an input (features) and a known output (label).

Goal: Learn a mapping function from inputs to outputs.

Examples of Supervised Learning Algorithms:
- Linear Regression: Predicts a continuous value (e.g., house price prediction)
- Logistic Regression: Predicts a category (e.g., spam or not spam)
- Decision Trees: Tree-like model of decisions
- Random Forest: Collection of decision trees (ensemble method)
- Support Vector Machines (SVM): Finds the best boundary between classes
- Neural Networks: Inspired by the human brain, used in deep learning
- K-Nearest Neighbors (KNN): Classifies based on nearest training examples

Applications:
- Email spam detection
- Image classification (cat vs dog)
- Medical diagnosis
- Stock price prediction
- Sentiment analysis

2.2 UNSUPERVISED LEARNING
--------------------------
In unsupervised learning, the model is given data WITHOUT labels. It must find hidden patterns or structures on its own.

Examples of Unsupervised Learning Algorithms:
- K-Means Clustering: Groups data into K clusters
- Hierarchical Clustering: Builds a tree of clusters
- DBSCAN: Density-based clustering
- Principal Component Analysis (PCA): Reduces the number of features
- Autoencoders: Neural networks that learn compressed representations

Applications:
- Customer segmentation (grouping customers by behavior)
- Anomaly detection (fraud detection)
- Topic modeling in documents
- Dimensionality reduction
- Recommendation systems

2.3 REINFORCEMENT LEARNING
---------------------------
In reinforcement learning, an agent learns to make decisions by interacting with an environment. The agent receives rewards for good actions and penalties for bad ones.

Key Components:
- Agent: The learner/decision maker
- Environment: The world the agent interacts with
- State: Current situation of the agent
- Action: What the agent does
- Reward: Feedback from the environment
- Policy: Strategy the agent uses to decide actions

Famous Algorithms:
- Q-Learning: Model-free RL algorithm
- Deep Q-Network (DQN): Q-Learning + Deep Neural Networks
- Policy Gradient Methods
- Actor-Critic Methods

Applications:
- Game playing (AlphaGo beat the world champion in Go)
- Robotics (teaching robots to walk)
- Self-driving cars
- Trading strategies in finance
- Resource management in data centers

2.4 SEMI-SUPERVISED LEARNING
-----------------------------
A combination of supervised and unsupervised learning. Uses a small amount of labeled data and a large amount of unlabeled data. Useful when labeling data is expensive or time-consuming.

CHAPTER 3: KEY MACHINE LEARNING CONCEPTS
==========================================

3.1 FEATURES AND LABELS
- Feature: An individual measurable property of the data (also called input variable or independent variable)
- Label: The output variable we are trying to predict (also called target variable or dependent variable)
- Feature Engineering: The process of selecting, creating, or transforming features to improve model performance

3.2 TRAINING, VALIDATION, AND TEST SETS
The dataset is typically split into three parts:
- Training Set (60-70%): Used to train the model
- Validation Set (15-20%): Used to tune hyperparameters and check for overfitting
- Test Set (15-20%): Used only at the end to evaluate final model performance

3.3 OVERFITTING AND UNDERFITTING
Overfitting:
- The model learns the training data TOO well, including noise and outliers
- Performs well on training data but poorly on new data
- Signs: High training accuracy, low test accuracy
- Solutions: More data, regularization (L1/L2), dropout, simpler model, cross-validation

Underfitting:
- The model is too simple to capture the patterns in the data
- Performs poorly on both training and test data
- Signs: Low training accuracy and low test accuracy
- Solutions: More complex model, more features, more training time, reduce regularization

3.4 BIAS-VARIANCE TRADEOFF
- Bias: Error from wrong assumptions. High bias → underfitting
- Variance: Error from sensitivity to small data fluctuations. High variance → overfitting
- The goal is to find the sweet spot with low bias AND low variance
- This is called the Bias-Variance Tradeoff

3.5 CROSS-VALIDATION
A technique to evaluate model performance more reliably.
- K-Fold Cross-Validation: Data split into K equal parts. Model trained K times, each time using a different part as the test set
- Stratified K-Fold: Ensures each fold has the same proportion of class labels
- Leave-One-Out (LOO): Each sample is used once as a test set

CHAPTER 4: MODEL EVALUATION METRICS
==========================================

4.1 FOR CLASSIFICATION PROBLEMS:
- Accuracy: (Correct predictions / Total predictions) × 100
- Precision: True Positives / (True Positives + False Positives)
- Recall (Sensitivity): True Positives / (True Positives + False Negatives)
- F1-Score: Harmonic mean of Precision and Recall = 2 × (Precision × Recall) / (Precision + Recall)
- Confusion Matrix: Table showing TP, TN, FP, FN
- ROC-AUC: Area Under the ROC Curve — measures model's ability to distinguish between classes

4.2 FOR REGRESSION PROBLEMS:
- Mean Absolute Error (MAE): Average of absolute differences between predicted and actual values
- Mean Squared Error (MSE): Average of squared differences
- Root Mean Squared Error (RMSE): Square root of MSE — in same units as target
- R² Score (Coefficient of Determination): How well the model explains variance in data. 1.0 is perfect.

CHAPTER 5: NEURAL NETWORKS AND DEEP LEARNING
==========================================

5.1 ARTIFICIAL NEURAL NETWORKS (ANN)
Inspired by the structure of the human brain. Consists of layers of interconnected nodes (neurons).

Components:
- Input Layer: Receives the input features
- Hidden Layers: Process and transform the data
- Output Layer: Produces the final prediction
- Weights: Parameters that are learned during training
- Bias: Additional parameter to shift the activation function
- Activation Function: Introduces non-linearity (ReLU, Sigmoid, Tanh, Softmax)

5.2 BACKPROPAGATION
The algorithm used to train neural networks.
1. Forward pass: Input goes through the network to produce a prediction
2. Calculate loss: Compare prediction to actual output using a loss function
3. Backward pass: Compute gradients using the chain rule
4. Update weights: Gradient Descent adjusts weights to reduce loss

5.3 COMMON ACTIVATION FUNCTIONS
- ReLU (Rectified Linear Unit): max(0, x) — most common in hidden layers
- Sigmoid: 1/(1+e^(-x)) — used in binary classification output layer
- Tanh: (e^x - e^(-x))/(e^x + e^(-x)) — outputs between -1 and 1
- Softmax: Converts outputs to probabilities for multi-class classification

5.4 TYPES OF DEEP LEARNING ARCHITECTURES
- CNN (Convolutional Neural Network): Used for image recognition and computer vision
- RNN (Recurrent Neural Network): Used for sequential data like text and time series
- LSTM (Long Short-Term Memory): Special RNN that solves vanishing gradient problem
- Transformer: Modern architecture used in NLP (powers ChatGPT, BERT)
- GAN (Generative Adversarial Network): Generates new data (images, music, text)
- Autoencoder: Learns compressed representations of data

5.5 GRADIENT DESCENT OPTIMIZERS
- Stochastic Gradient Descent (SGD): Updates weights after each sample
- Mini-Batch Gradient Descent: Updates weights after each mini-batch
- Adam (Adaptive Moment Estimation): Most popular optimizer, adapts learning rate
- RMSprop: Adapts learning rate based on recent gradients
- Adagrad: Adapts learning rate based on historical gradients

CHAPTER 8: ML PIPELINE (END-TO-END)
==========================================

Step 1: Problem Definition
- Define the business problem
- Determine if it's classification, regression, or clustering

Step 2: Data Collection
- Gather data from databases, APIs, web scraping, surveys

Step 3: Data Preprocessing
- Handle missing values (imputation or removal)
- Remove duplicates
- Handle outliers
- Encode categorical variables (Label Encoding, One-Hot Encoding)

Step 4: Exploratory Data Analysis (EDA)
- Understand distributions and relationships
- Visualize data with plots and charts
- Find correlations between features

Step 5: Feature Engineering
- Create new features from existing ones
- Select relevant features (feature selection)
- Scale features (Normalization, Standardization)

Step 6: Model Selection
- Choose appropriate algorithm based on problem type and data size
- Consider model complexity and interpretability

Step 7: Model Training
- Train model on training data
- Tune hyperparameters using validation set

Step 8: Model Evaluation
- Evaluate on test set using appropriate metrics
- Compare with baseline model

Step 9: Model Deployment
- Package model as an API (Flask, FastAPI)
- Monitor model performance in production
- Retrain when performance degrades (model drift)

QUICK REVISION — KEY FORMULAS
==========================================

Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1 Score = 2 * (Precision * Recall) / (Precision + Recall)
MSE = (1/n) * Σ(actual - predicted)²
RMSE = √MSE
MAE = (1/n) * Σ|actual - predicted|
